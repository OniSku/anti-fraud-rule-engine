# 🛡️ Anti-Fraud Business Rule Engine (API Mock)

Асинхронный движок бизнес-правил (Rule Engine), разработанный для валидации финансовых транзакций, аккаунтов и устройств. Проект демонстрирует архитектурный подход к созданию систем антифрода и комплаенса, где правила проверки не зашиты в код ("hardcoded"), а управляются динамически.

## 🎯 Бизнес-задача
Необходимо было реализовать ядро системы, которое позволяет администраторам на лету создавать, изменять и удалять правила (например, лимиты по суммам или блокировки по IP/странам), а системе — проверять входящие бизнес-объекты на соответствие этим правилам перед сохранением в базу.

## 🛠 Технологический стек и Архитектура
* **Язык**: Python 3.10+ (с использованием `asyncio` для имитации сетевых задержек БД).
* **Валидация**: `Pydantic v2` — строгая типизация и проверка входящих JSON-данных перед их передачей в ядро.
* **Паттерны проектирования**: 
  * **Repository Pattern**: Слой работы с данными (`src/database.py`) изолирован от бизнес-логики. Сейчас используется In-Memory хранилище, но переход на SQLAlchemy/PostgreSQL потребует изменения только одного класса, не затрагивая сервис проверок.
  * **Dependency Injection**: Движок правил работает через интерфейсы, принимая подготовленные данные.

## ⚙️ Как работает движок (Логика)
1. Администратор задает правило: `Target: Transaction, Field: amount, Operator: >, Value: 10000`.
2. Пользователь инициирует транзакцию на `15000`.
3. API прогоняет объект через валидацию `Pydantic`.
4. `RuleEngineService` динамически подтягивает активные правила для типа `Transaction`.
5. Движок применяет оператор и блокирует транзакцию, возвращая массив `errors` с описанием нарушенного правила.
6. Легитимные объекты (прошедшие все проверки) отправляются в DAL (Data Access Layer) для сохранения.

## 🚀 Запуск демонстрации
Проект включает в себя скрипт-симулятор, который демонстрирует работу API, генерацию правил и фильтрацию объектов в реальном времени.

1. Установите зависимости: `pip install -r requirements.txt`
2. Запустите симуляцию: `python -m src.main`
