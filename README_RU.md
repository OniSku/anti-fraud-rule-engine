# Anti-Fraud Rule Engine

Асинхронный движок бизнес-правил, разработанный для валидации финансовых транзакций, аккаунтов и устройств. Проект демонстрирует архитектурный подход к созданию систем антифрода и комплаенса, где правила проверки управляются динамически, а не зашиваются в код.

## Описание

Ядро, позволяющее администраторам в реальном времени создавать, изменять и удалять правила валидации (например, лимиты по суммам или блокировки по IP/странам). Входящие бизнес-объекты проверяются на соответствие активным правилам перед сохранением в базу данных.

## Ключевые возможности

- **Динамическое управление правилами**: Создавайте, обновляйте и удаляйте правила валидации без развертывания кода
- **Поддержка множественных объектов**: Валидация транзакций, аккаунтов и устройств с помощью унифицированного движка правил
- **Валидация в реальном времени**: Немедленное применение правил к входящим запросам
- **Типобезопасность**: Строгая валидация с помощью схем Pydantic
- **Асинхронная архитектура**: Имитация задержек ввода-вывода базы данных для реалистичного тестирования производительности
- **Repository Pattern**: Четкое разделение между доступом к данным и бизнес-логикой
- **Поддержка операторов**: Множество операторов сравнения (>, <, ==, !=, in)

## Принцип работы

1. **Создание правил**: Администратор определяет правило (например, `Цель: Transaction, Поле: amount, Оператор: >, Значение: 10000`)
2. **Валидация объекта**: Пользователь инициирует транзакцию на сумму `15000`
3. **Валидация схемы**: API проверяет данные через Pydantic
4. **Применение правил**: `RuleEngineService` динамически загружает активные правила для типа `Transaction`
5. **Принятие решений**: Движок применяет операторы и блокирует транзакцию, возвращая массив `errors` с нарушениями правил
6. **Сохранение данных**: Легитимные объекты (прошедшие все проверки) отправляются в DAL для сохранения

## Установка и настройка

### Предварительные требования

- Python 3.10+
- pip

### Переменные окружения

| Переменная | Описание | Обязательная |
|------------|----------|--------------|
| `ADMIN_SECRET_KEY` | Секретный ключ для административных операций | Да |
| `DEBUG` | Включить режим отладки | Нет (по умолчанию: True) |

### Быстрый старт

1. Клонируйте репозиторий:
```bash
git clone https://github.com/OniSku/anti-fraud-rule-engine.git
cd anti-fraud-rule-engine
```

2. Установите зависимости:
```bash
pip install -r requirements.txt
```

3. Настройте окружение:
```bash
cp .env.example .env
# Отредактируйте .env с вашими значениями
```

4. Запустите демонстрацию:
```bash
python -m src.main
```

Симуляция продемонстрирует:
- Загрузку активных правил из базы данных
- Обработку различных сценариев валидации транзакций и аккаунтов
- Динамическое создание правил администраторами
- Применение правил в реальном времени и принятие решений

## Паттерны проектирования

- **Repository Pattern**: Слой доступа к данным изолирован от бизнес-логики, что обеспечивает легкую миграцию базы данных
- **Dependency Injection**: Движок правил работает через интерфейсы, принимая подготовленные данные
- **Валидация схем**: Pydantic обеспечивает типобезопасность и целостность данных перед обработкой

## Примеры правил

- Блокировка транзакций свыше $10,000
- Ограничение аккаунтов из определенных стран
- Блокировка недоверенных устройств
- Проверка возраста для создания аккаунта
- Кастомная бизнес-логика через настраиваемые операторы
